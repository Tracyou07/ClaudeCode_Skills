# Code Review

多代理并行 PR 审查，带置信度评分过滤。

## 调用

```
/code-review
```

## 工作流程

1. 检查 PR 是否需要审查（跳过已关闭/草稿/已审查的）
2. 收集相关 CLAUDE.md 规范文件
3. 启动 5 个并行代理分别审查：
   - Agent 1: CLAUDE.md 合规性
   - Agent 2: 明显 bug 扫描
   - Agent 3: git blame 历史上下文
   - Agent 4: 历史 PR 评论参照
   - Agent 5: 代码注释合规性
4. 每个问题独立评分 0-100
5. 过滤 < 80 分的问题，只发布高置信度反馈

## 置信度评分

| 分数 | 含义 |
|------|------|
| 0 | 误报 |
| 25 | 可能真实，未验证 |
| 50 | 真实但较小 |
| 75 | 高置信度，验证通过 |
| 100 | 完全确认 |

## 要求

- GitHub CLI (`gh`) 已安装并认证
- Git 仓库有 GitHub remote
